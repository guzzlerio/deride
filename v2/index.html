derideMocking that wraps, not patches.

TypeScript-first test doubles for frozen objects, sealed classes, and any coding style — with matchers, spies, and first-class fake timers.

Get started
Why deride?
View on GitHub
🧊

Works with frozen objects

Composition-based. Wraps your real objects instead of mutating them, so `Object.freeze`, sealed classes, and prototype-based code just work.

🔍

Matcher-first assertions

A composable match.* namespace — any, string, number, objectContaining, gte, regex, allOf — reusable across setup and expectations.

🧠

Spy + expect, split by purpose

expect.* throws on mismatch for assertions. spy.* reads call history as data — feed it forward, snapshot it, print it.

🧵

Cross-mock call ordering

inOrder(a.spy.x, b.spy.y, …) asserts first-call sequence; inOrder.strict enforces no interleaved extras.

🧪

Integrates where you test

deride/vitest and deride/jest ship first-class toHaveBeenCalled* matchers. deride/clock is a dependency-free fake timers helper.

📐

Type-safe by default

Setup methods are constrained to your method signatures. Full generic inference. as any escape hatch for testing error paths.

🏗️

Auto-discovers class methods

stub(MyClass) walks the prototype chain. stub.class<typeof C>() mocks constructors with expect.constructor and per-instance setup.

🔄

Snapshot and restore

Capture any mock's behaviours and call history, mutate freely, roll back to the snapshot. sandbox() fans out across every registered mock at once.

🪶

Zero runtime dependencies

Ships with only debug as a runtime dep. Sub-paths (clock, vitest, jest) tree-shake cleanly. Under 40 KB packed.


At a glance ​

typescript
import { stub, match } from 'deride'

interface Database {
  query(sql: string): Promise<unknown[]>
  findById(id: number): Promise<unknown>
}

const mockDb = stub<Database>(['query', 'findById'])
mockDb.setup.query.toResolveWith([{ id: 1, name: 'alice' }])
mockDb.setup.findById.when(match.gte(100)).toRejectWith(new Error('not found'))

const result = await mockDb.query('SELECT * FROM users')

mockDb.expect.query.called.once()
mockDb.expect.query.called.withArg(match.regex(/FROM users/))
mockDb.expect.findById.called.never()

No monkey-patching. mockDb is a wrapper around a fresh object. Your real Database class is untouched. That's why deride works on frozen objects, sealed classes, and any coding style.
